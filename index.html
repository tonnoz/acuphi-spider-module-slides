ACUSIDEA provides services and products for

more than 15 million of final users

Developed from ACUSIDEA  to handle fetching, operations and QA for the data-flows exchanging between the selling companies and the distribution companies in the Italian energy market.
Among the managed flows there are the PROCESSES,  but also  demographic  information or measurement of energy consumption.

Who are the users of ACUPHI?
Multi-utilities and other big actors in the Italian energy market.
ACUPHI provides functionalities in order to manage the data acquired from distribution portals and provide an high quality data elaborations. Moreover, offers tools for data-inspections and charting.

Motivations

Automation of sending and retrieval of data-flows in the Italian-Energy market (practically files exchange on web-portals)

Specific context

"PROCESS"

Management of operations activatable on a Point Of Delivery (eg. switch-in)

  1. Subdivide the requests of a determinate service, eg. block of supply for arrearage and grouping them by company.
  2. Insert these requests in a specific file with a predetermined template, given by the distribution company.
  3. Browse manually the web-portal of the distributor and get to the upload page.
  4. Upload the file with the requests.
  5. Repeat the steps from 2 to 4 until all the requests for all the distributors are submitted.

How to automate this workflow?

Spider?

Spider / web-crawler

an Internet bot which systematically browses the internet, typically for the purpose of Web indexing or data extraction.

Headless browser

a web browser without a graphical user interface drivable through an API.

Example

  • Free the employee from the manual work in dealing with the browsing of every single web portal of energy distribution and from the management of the requests, with the introduction of a web-tool, usable from a simple graphic user interface.

Objectives

  • Developing an architecture of web-crawlers in Java® capable of handling all the necessary spider operations.
  • Developing and testing of the different web-crawlers for the different processes and web portals, using an Java API for the headless browsing and an Oracle database for the instructions storing.

How?

Operations that a Spider can do

  • HTTP GET, HTTP POST 
  • Analyze the DOM
  • Interact with elemets of the DOM (es. fill di text field, select option, etc..)
  • Download triggered by an event (es click o get/post) 
  • Upload triggered by an event (click submit button o POST multipart)
  • Access to a secure session via certificate X.509
  • Emulate REST calls to a remote WebService (SOAP messages)
